translate

Unofficial Description

The CanvasRenderingContext2D.translate() method of the Canvas 2D API shifts the coordinate axis of the canvas by the X and Y values provided.

Why Would I Use This?

translate() allows you to shift the entire coordinate grid of the canvas context.

So, instead of changing the x and y coordinates of many commands you can just shift the coordinate grid and the drawing will appear in a different location. Here's an example of copying a drawing of a house:

Editor (write code below)
var canvas = document.getElementById('translate_interactive1'); var context = canvas.getContext("2d"); context.clearRect(0, 0, 600, 360); for (var i=0; i<3; ++i) { context.save(); context.translate(160*i, 25*i); context.beginPath(); context.moveTo(25, 100); context.lineTo(175, 100); context.lineTo(175, 250); context.lineTo(25, 250); context.lineTo(25, 100); context.lineTo(100, 25); context.lineTo(175, 100); context.moveTo(50, 125); context.lineTo(75, 125); context.lineTo(75, 150); context.lineTo(50, 150); context.lineTo(50, 125); context.moveTo(85, 250); context.lineTo(85, 190); context.lineTo(115, 190); context.lineTo(115, 250); context.stroke(); context.restore(); }

Example

Here's a basic example where the same fillRect() command draws in different areas due to translate()

var canvas = document.getElementById('translate_example1'); var context = canvas.getContext("2d"); context.save(); context.clearRect(0, 0, 600, 360); context.fillRect(50,175,100,100); context.translate(125,-125); context.fillRect(50,175,100,100); context.restore();
